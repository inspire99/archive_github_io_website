    TECHNO PARADIGMS: SVELTE : MOTIVATION




    Every now and then, a new javascript framework comes along, makes developer's hands dirty (literally).

    So, why was svelte js created?

    What is this new approach to building user interfaces? What is this new paradigm?

    Why is Svelte js fast?

    From Svelte website,

    Svelte converts your app into ideal JavaScript at build time, rather than interpreting your application code at run time. This means you don't pay the performance cost of the framework's abstractions, and you don't incur a penalty when your app first loads.

    What is the meaning of svelte? -  From Oxford dictionary, Svelte is an adjective attributed to a person, which means - slender and elegant - https://www.lexico.com/en/definition/svelte

    So, what makes it thin / elegant - or how does the author claim that it is why it is 'svelte' ?

    The reason being: the bundle size is very small.

    But, if it is small, what about performance?

    Svelte js applications seem to be fast too.

    So, how does svelte js ensure that bundle size is very small?

    And how does svelte js ensure that it runs fast too?

    One difference with other frameworks like: Vue/ React is that Svelte does not have concept called : virtual dom. So, what do these vdom-enabled frameworks do? Ultimately, the dom of the webpage needs to be updated, in order to be provided with some changes in the dom tree. These frameworks deal finally with the DOM of the webpage. For this , they construct a virtual dom. Now, the framework can determine which parts of this virtual dom needs change , based on the updates to the components corresponding to the vdom. Then finally, the updates are pushed to the real-dom of the webpage.

    Svelte js skips the vdom part, it is able to deal directly with the real dom and pushes the updates 'quickly'. So, how does svelte js do, which React/Vue was not able to do? And can svelte js do it quickly?

    So, since Svelte js is a compiler, it compiles the written code into a final vanilla js code. So, the bundle doesnot contain any unnecessary dependencies at all. Imagine there are dependent js files , which contains code, which is never used within your application, literally they are dead code. But still they will be bundled with your application. So, what svelte does is that, it does not result with any unnecessary dependency.

    What all happens before a webpage is finally rendered?

    With Svelte js, you can imagine that, finally what is present within the browser/ client is pure javascript. So, when you open a webpage, pure javascript renders the webpage.

    Whereas, with other frameworks such as React/ Vue, the vdom builder code needs to be loaded. Then the particular library performs the rendering. ( and not pure vanilla js )

    ( If you are a backend java programmer, you might have heard of Quarkus - which does the same thing - improving the startup of the application - during build time itself, all the unnecessary dependencies are removed out, so what finally runs within the VM is a very minimal sized app . So, within cloud native environment, the application density can be more - more applications can run within the same machine.)

    So, I am actually curious about Svelte compiler.


    Svelte Compiler
    ============

    The code for Svelte Compiler is at this Github link .

    As we can see, it uses ACORN for parsing <script> tags.
    As we can see, it uses CSS-TREE for parsing <style> tags.


    As with any compiler, parsing is the first step.
    Once parsing is done, Abstract Syntax Tree is prepared out of the code.
    Then code generation is the final step.
