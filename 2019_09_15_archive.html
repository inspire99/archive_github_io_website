Java - How to print to Standard Output




Let us now write a program in Java, which prints "Hello World" onto the console.

 Code:

public class HelloWorld {

    public static void main(String[] args) {
        System.out.println("Hello World");    
    }
    
}

If you want to try this out, you can copy the code and run this online

 Or in your local system, you can install VSCode IDE, install Java Extension Pack, ensure you have JDK installed locally. And from VSCode, you can run the code.
Code Explanation
  • HelloWorld is the class 
    • It is public
  • The main method is invoked, when we run this code.
  • System is another class, which is like a utility class.
    • It provides access to system console
    • We can use println method to print to console.

What is new with Android 10







Android 10 is officially available .

 Android 10 has brought new set of features across following aspects in Android ecosystem.
  • Usability
  • Security
  • Enterprise
  • Privacy
  • Location
  • Family

  1. Live Caption
    1. This is an accessibility feature. It will automatically caption media playing, including videos, podcasts, audio messages, “across any app.”
  2. Smart Reply
    1. Based on content in message received, it can help you to take action, while replying to that message. If address comes in message, then google maps button will be enabled, so you can take action to go right into google maps.
  3. Sound Amplifier
    1. Imagine talking in a busy room — just plug in your headphones and hear everything more clearly. This is basically an app - https://play.google.com/store/apps/details…
  4. Gesture Navigation
    1. Within smartphone, using touch/ gestures do easy navigation. Go backwards and forwards, pull up the homescreen, and swipe up to see your open apps.
  5. Dark Theme
    1. From Android 10, you can enable dark theme. It saves battery and good for eyes too.
  6. Privacy Controls
    1. You can decide how and when your data is being used.
  7. Location Controls
    1. You will become aware if some app needs location usage, then you can control it.
  8. Security Updates
    1. Security Updates will be applicable as soon as they arrive.
  9. Focus Mode
    1. Using an app, you can go to focus mode. Do some settings, so that, you do not get distracted.  Its an app - Digital Wellbeing - https://play.google.com/store/apps/details…
  10. Family Link
    1. Set digital ground rules and help make healthy habits. Help guide your kids as they learn, play, and explore online. You can set screen time limits, view app activity, manage apps and content restrictions, and see where they are.

A thing about work profile on Android.

What is a work profile on Android device?

Imagine you are working in an organization. The organization decides to leverage an opportunity with Android mobile device. As the estimates show, more people use android device. And organization can use this, so that employees can work on the go. By this, the organization can decide to create work apps. Android apps which cater to their organization. For example, some employee who is in charge of inventory (Inventory Manager) for a retail store, might get notification that, there is some item whose stock has gone low. That employee can take action by calling up the resource who is responsible for replenshing the stock or the employee can make orders to ensure that stock is refilled. This entire scenario is possible, on the go, enabled by usage of Android by the employee and by usage of Android ecosystem by the organization. This is just one example, there could be many use-cases which cater to enterprise scenarios where Android / mobile phones could be leveraged.

Now that, you have read above paragraph, head over to Android for employees

Now that, we have read a bit about work profile, we might face some questions, as to how onboarding would be done and how offboarding would be done.

  • We can as well think of the entire lifecycle for that work app, in the context of that employee and that organization, as jotted below:
    • For example, once a new employee is added to the organization, how easy it is to roll out the work apps on to the employee's mobile.
    • Once an employee leaves the organization, the work apps should be uninstalled or erased completely, as organization-specific confidential data still rests within the mobile device.
    • While the employee works with the organization, how is the data privacy of work apps ensured ? For example, there should be no way for the user to transfer data from work app to personal app ( WhatsApp ? )
    • How are the updates rolled out to the employees? 
    • Is it a BYOD use-case? Can the employee bring their own device? So, they can use personal apps alongside work apps.
While reading about the above use-cases, I found Scalefusion which provides solutions in this domain. Head over to https://scalefusion.com/features if you are interested.
