What is bytecode manipulation

https://in.pinterest.com/pin/375487687679904797/
What are the frameworks used almost daily in our Java projects ?

 ( Answer might vary depending on usage/ experience )
  • Spring (dependency injection, aop)
  • Hibernate ( orm )
  • Mockito ( mocking objects for testing )
  • Profilers ( finding performance leaks , timing method executions)
  • Application Performance Monitoring. 

Are you aware that these frameworks (Spring, Hibernate, Mockito) heavily depend on bytecode manipulation?


What is bytecode manipulation ?
  • Modifying content of class
    • add fields
    • add methods
    • modify fields

Let us be clear that bytecode is nothing but the .class file. So, there is a possibility of editing this bytecode. 

Is it possible to only edit existing bytecode (shh shh ) ?
Or is it capable enough to introduce new byte code? Meaning, new class file (shh shh ) ?

Some questions to be asked is:
  • when does bytecode manipulation happen? Or at what stage does it happen?
    • At compile time? ( at compile time, do we have bytecode yet ? )
    • At run time?
    • At load time ?
      • When JVM actually loads the class?

Some of the frameworks enabling bytecode manipulation are:

  • ASM
  • AspectJ
  • BCEL
  • Byte Buddy
  • CGLIB
  • Cojen
  • Javassist
  • Serp



For example, let us consider a use-case where we want to do some logging when a method is called. This can be achieved using aspect oriented programming .

package org.simpleaop.aspect;
//..import statements
 
@Aspect
public class LoggingAspect {
       
       @Before("execution(public String getName())")
       public void beforeAdvice(){
             System.out.println("Before advice. Runs before the function called");
       }
       //...other advices
}

Code courtesy: https://www.developer.com/java/implementing-aop-in-spring.html

When we run the code, we would get output where in code from LoggingAspect will get executed.l
Behind the scenes, AspectJ is able to manipulate bytecode such that, the beforeAdvice method gets invoked.
